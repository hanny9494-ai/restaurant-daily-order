# 项目情况书（Project Status）

- 项目名称：Jify Culinary R&D Copilot
- 负责人：Jeff
- 最近更新时间：2026-03-01
- 当前阶段：V2 知识库重构（全量蒸馏进行中）

## 1. 项目目标
- 构建可用于厨师研发工作的智能系统，不只是检索答案，而是支持从感官意图到技术实现的完整推理链。
- 以高质量厨书为核心数据源，建立可验证、可扩展、可持续增量更新的知识引擎。

## 2. 当前进展
- Dify 工作流已从单一路径升级为多意图路由（完整开发、单组件、技术问答、优化修改、灵感探索）。
- 知识库服务层保持 6 库结构：science / ingredients / techniques / recipes / sensory / troubleshooting。
- V2 蒸馏管线已运行：49 本书，14,343 chunks，支持断点续跑与 coding-plan/payg 自动切换。
- V2 项目文档体系已建立：状态、backlog、周计划、runbook、推送指南。

## 3. 本周重点
- 跑完 49 本全量蒸馏并导出质量统计。
- 抽样验证标注质量（主次标签、证据引用、适用边界）。
- 产出 Dify 导入映射方案，准备灰度切换。

## 4. 风险与阻塞
- 首个 chunk 可能出现长延迟，易被误判为进程卡死。
- 若缺少抽样质检，标注质量可能漂移。
- 费用风险：触发 payg 频率过高会抬升成本。

## 5. 下一步计划
- 完成全量蒸馏后，派生 4 个内部功能层：Core-Science / Execution-Playbook / Failure-Atlas / Inspiration-Lab。
- 基于回归问题集比较旧库与新库效果，达标后执行 Dify 灰度迁移。
- 为新书建立标准增量入库流程（OCR -> 蒸馏 -> 归档 -> 派生层 -> Dify映射）。
